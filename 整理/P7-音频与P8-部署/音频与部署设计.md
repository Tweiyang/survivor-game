---

```mermaid
mindmap
```

---

## ⚡ 设计技巧

| 技巧 | 说明 | Unity 对应 |
|------|------|-----------|
| **单例** | 全局一个 AudioManager | `AudioManager.instance` |
| **淡入淡出** | `linearRampToValueAtTime` | `AudioSource.DOFade` |
| **自动播放限制** | 首次用户交互后 `resume()` | 不需要（Unity 无此限制） |
| **音量节点图** | GainNode 层级控制 | `AudioMixer` |

# P8 — Web 部署设计

> GitHub Pages 前端 + Render 后端，环境自适应，优雅降级。

---

## 🧠 设计思维导图

```mermaid
mindmap
  root((P8 Web 部署))
    前端托管
      GitHub Pages
        git push 即部署
        .nojekyll 禁 Jekyll
        静态文件直出
    后端托管
      Render
        免费 Web Service
        自动从 GitHub 部署
        render.yaml Blueprint
        /health 健康检查
    环境自适应
      NetworkManager
        _resolveServerUrl
        meta 标签优先
        localhost 检测
        wss 自动判断
    降级策略
      后端不可用
        单人仍可玩
        5s 超时提示
        错误自动消失
```

---

## 🌐 部署架构

```mermaid
graph LR
    Dev["开发者<br/>git push"] --> GH["GitHub<br/>仓库"]
    GH --> Pages["GitHub Pages<br/>前端静态"]
    GH --> Render["Render<br/>后端服务"]
    
    User["玩家"] --> Pages
    Pages -->|WSS| Render
    
    style Pages fill:#2ECC71,color:#fff
    style Render fill:#3498DB,color:#fff
```

---

## ⚡ 设计技巧

| 技巧 | 说明 |
|------|------|
| **.nojekyll** | 禁止 GitHub 用 Jekyll 构建，保留原始 JS 文件 |
| **meta 标签配置** | `<meta name="game-server" content="wss://...">` 不改代码换服务器 |
| **CORS** | 服务端允许 `*.github.io` 跨域 |
| **环境检测** | `localhost` → `ws://`，其他 → `wss://` |
| **超时保护** | 5 秒连接超时，不让玩家无限等待 |
